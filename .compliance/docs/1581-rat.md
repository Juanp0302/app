## Decisiones de diseño relevantes para privacidad

- **Segregación multi-tenant:** cada cliente tiene un `cliente_id`; todas las consultas filtran por este ID. Implementado en lib/clientes.ts y todas las API routes.
- **Control de acceso por roles:** admin, cliente, superadmin con verificación en middleware y API routes.
- **No se procesan datos sensibles** de los titulares personas naturales (Art. 6 Ley 1581).
- **No hay usuarios menores de edad** en la plataforma.
